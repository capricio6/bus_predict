Netlify로 외부 공개 웹앱 만들기

결과
- 배포가 끝나면 https://원하는이름.netlify.app 같은 공개 주소가 생깁니다.
- 휴대폰은 PC 없이 그 주소로 바로 접속합니다.
- 홈 화면에 추가하면 앱처럼 사용할 수 있습니다.

왜 Netlify도 가능한가
- 화면 파일은 정적 사이트로 배포합니다.
- 공공 API 호출은 Netlify Functions가 대신 합니다.
- API 키는 BUS_API_KEY 환경변수에 넣어서 휴대폰 화면 코드에 노출하지 않습니다.

1. GitHub에 올리기
- bus_predictor_app 폴더 전체를 GitHub 저장소에 올립니다.
- config.env 파일은 올리지 마세요. .gitignore에 제외되어 있습니다.

2. Netlify에서 Site 만들기
- https://www.netlify.com 에 로그인합니다.
- Add new site > Import an existing project를 선택합니다.
- GitHub 저장소를 연결합니다.
- Base directory가 필요하면 bus_predictor_app 폴더를 지정합니다.

3. 빌드 설정
- Build command: npm install && npm run build
- Publish directory: .
- Functions directory: netlify/functions

4. 환경변수 등록
- Site configuration > Environment variables에서 BUS_API_KEY를 추가합니다.
- 값에는 공공데이터 API 키를 넣습니다.

5. 배포 후 사용
- Netlify가 만든 공개 URL을 휴대폰에서 엽니다.
- Android Chrome: 메뉴 > 홈 화면에 추가
- iPhone Safari: 공유 버튼 > 홈 화면에 추가

주의
- 공공 API 일일 호출 제한이 있으므로 새로고침을 너무 자주 하면 제한에 걸릴 수 있습니다.
- Netlify 무료 플랜의 Functions 사용량 제한도 있습니다.
